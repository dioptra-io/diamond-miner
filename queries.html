ClickHouse IPv6 type.

FixedString = 'FixedString'¶

Native FixedString(16) representation. Fastest with clickhouse-driver as it will return directly a byte string.

IPv6 = 'IPv6'¶

IPv6 stored as FixedString(16) internally. Slow with clickhouse-driver as it will call ipaddress.IPv6Address() on each address.

IPv6NumToString = 'IPv6NumToString'¶

Human string representation of the IPv6. With clickhouse-driver each address will be decoded as Python string.

String = 'String'¶

String representation of the internal FixedString. With clickhouse-driver each address will be decoded as Python string.

class diamond_miner.queries.GetLinksFromView(addr_type: diamond_miner.queries.query.AddrType = AddrType.IPv6, round_eq: Optional[int] = None, ignore_invalid_prefixes: bool = True, optimize_aggregation_in_order: bool = False)[source]¶

Compute the links from the flows view. This returns one line per (flow, link) pair.

We do not emit a link in the case of single reply in a traceroute. For example: * * node * *, does not generate a link. However, * * node * * node', will generate (node, *) and (*, node').

We emit cross-rounds links. For example if flow N sees node A at TTL 10 at round 1 and flow N sees node B at TTL 11 at round 2, we will emit (1, 10, A) - (2, 11, B).

We assume that there exists a single (flow, ttl) pair over all rounds. TODO: Assert this?

If round_eq is none, compute the links per flow, across all rounds. Otherwise, compute the links per flow, for the specified round. This is useful if you want to update a links table round-by-round: such a table will contain only intra-round links but can be updated incrementally.

>>> from diamond_miner.test import url
>>> links = GetLinksFromView().execute(url, "test_nsdi_example")
>>> len(links)
58
optimize_aggregation_in_order: bool = False¶

Necessary to avoid excessive memory usage when inserting all the links in one batch.
